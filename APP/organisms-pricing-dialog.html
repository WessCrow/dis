Organismo Interface Principal

Pricing Dialog

Diálogo modal acoplado à navegação superior para gestão rápida de planos (Mensal/Anual) e upgrades de tokens.

UX Writing & Identidade

Voz: Direta, objetiva e transparente.

Tom: Confiante (sem ansiedade de conversão). O foco é mostrar o limite exato de recursos sem culpar o utilizador por "bater no teto".

Regra de Ouro: Nunca arredondar preços para cima. Exibir a moeda separada do valor para alinhamento ocular perfeito.

Planos e tokens

Matriz FEER (Technical Rigor)

Fluxos (Flows)

Passo Ação do Usuário Resposta do Sistema
1 Clica no Trigger .lab-tokens__open-pricing O dialog abre no topo direito (fixo no viewport), foco salta para o fechar.
2 Seleciona o período (Anual) A posição do selector (Knob) anima e os preços interpolam os valores.
3 Escolhe um novo plano O anel (ring) de contorno anima em volta do plano, o rádio preenche. O plano atual fica retido (disabled) se o utilizador já o tiver.

Estados de UI (States)

Estado Descrição Visual
aria-pressed="true" Borda do plano transita para branco var(--text) e aplica ring externo.
[disabled] Opacidade de 50%, evento de clique ignorado (para o plano atualmente subscrito).
Loading (Transição) Interpolação JavaScript pura para incrementar os cêntimos visualmente (`.lab-pricing__amount--flow`).

Acessibilidade (POUR)

Princípio Aplicação (HTML/CSS)
Perceptível Cores no standard APCA, sem dependência exclusiva de cor para estado ativo (usa ring border e radio button).
Operável Teclado funciona nos botões de período (Tab) e no botão de fechar. Foco mantido dentro do dialog via trap de JS.
